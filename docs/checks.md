# Available Checks

## Complexity

Analyzes code complexity metrics:

- Cyclomatic complexity
- Nesting depth
- Function length
- Parameter count

## Style

Checks code style conventions:

- Line length
- Trailing whitespace
- Print statement usage

## Security

Detects security vulnerabilities:

- Dangerous function usage (eval, exec, etc.)
- SQL injection patterns
- Path traversal
- Command injection

## Performance

Identifies performance bottlenecks:

- Nested loops
- Large memory allocations
- Slow module-level imports

## Documentation

Ensures documentation coverage:

- Module docstrings
- Function docstrings
- Class docstrings
- Docstring length

## Naming

Validates naming conventions:

- PascalCase for classes
- snake_case for functions and variables

## Imports

Analyzes import organization.

## Duplication

Detects duplicated code patterns.

## Typing

Checks type annotation usage:

- Return type annotations
- Parameter type annotations
